# Source Register and Design Influence

## Control Hierarchy

1. Current project instructions
2. Current `gantt.xlsx`
3. Accepted workflow contracts and architecture documents
4. Frozen release/benchmark evidence
5. Market and research sources

Research is advisory. It does not silently change scope, acceptance criteria, or release claims.

## Research Papers

### Agents' Last Exam

Primary influence:

- authentic professional workflows;
- economically meaningful end-to-end deliverables;
- task/agent/environment separation;
- artifact-based verification;
- gate-and-score evaluation;
- deterministic checks wherever possible;
- public/private or development/held-out task separation.

Reference:

- https://arxiv.org/abs/2606.05405

### OpenClaw-RL: Train Any Agent Simply by Talking

Later influence:

- next-state signals from tool output, terminal errors, tests, GUI changes, and corrections;
- separate process and outcome evidence;
- asynchronous trace/signal architecture.

Control note:

- deferred until a stable package workflow and sufficient preserved traces exist.

Reference:

- https://arxiv.org/abs/2603.10165

### RLAnything

Later influence:

- integrated outcome and step-wise signals;
- reward-model calibration;
- environment/task adaptation based on observed failure patterns.

Control note:

- do not attempt RL before valid tasks, stable evaluators, traces, and reward calibration.

Reference:

- https://arxiv.org/abs/2602.02488

## Source Maintenance Rules

- Record source date and version for any architecture-changing claim.
- Prefer official documentation and primary papers.
- Re-verify fast-changing model/tool claims before major decisions.
- Do not copy confidential platform or employer material into the source register.
- Record a Decision Register entry when a source materially changes project scope or architecture.
